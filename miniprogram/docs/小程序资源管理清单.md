# 叨叨房车小程序资源管理清单

**文档版本**: v1.0.0 | **创建时间**: 2025-12-27 | **维护者**: 叨叨房车技术团队

**文档类型**: 资源管理清单 | **适用对象**: 管理端开发人员、产品经理、运营人员

---

## 📋 文档说明

本文档列出了叨叨房车小程序端所有需要通过管理端动态管理的资源内容。这些资源目前在小程序前端代码中是硬编码的,需要通过管理端提供后台管理界面,实现实时替换和更新。

**目标**: 指导管理端创建"小程序资源管理"菜单,实现所有小程序内容的动态化管理。

---

## 🎯 资源管理总览

### 资源分类统计

| 资源类别 | 子类数量 | 优先级 | 说明 |
|---------|---------|--------|------|
| 图片资源 | 8类 | ⭐⭐⭐⭐⭐ | Banner、图标、背景图等 |
| 文本内容 | 6类 | ⭐⭐⭐⭐⭐ | 公告、协议、帮助文档等 |
| 配置项 | 5类 | ⭐⭐⭐⭐ | 会员权益、服务菜单等 |
| 业务数据 | 4类 | ⭐⭐⭐ | 特惠套餐、优惠券等 |

### 管理端菜单建议结构

```
小程序资源管理
├── 图片资源管理
│   ├── 轮播图管理

```

## 📸 一、图片资源管理

### 1.1 首页轮播图 (Banner)

**当前状态**: 硬编码在 `pages/index/index.vue`

**硬编码示例**:
```typescript
const banners = ref<Banner[]>([
  { id: '1', image: '/static/场景推荐2.jpg', link: '' },
  { id: '2', image: '/static/优惠政策.jpg', link: '' }
]);
```

**管理需求**:

- **功能**: 管理首页顶部轮播广告图
- **字段要求**:
  - 图片URL (必填) - 支持上传或输入URL
  - 跳转链接 (可选) - 点击后跳转的页面路径
  - 排序权重 (必填) - 控制显示顺序
  - 是否启用 (必填) - 开关控制
  - 生效时间 (可选) - 开始和结束时间
- **图片规格**: 750×320px, JPG/PNG, 最大500KB
- **数量限制**: 建议3-5张
- **API接口**: `GET /api/v1/miniprogram/banners`

**使用页面**:

- pages/index/index.vue (首页)

### 5.2 技术实现方案

**后端API设计**:
```
GET  /api/v1/miniprogram/resources/{type}     # 获取资源列表
POST /api/v1/miniprogram/resources            # 创建资源
PUT  /api/v1/miniprogram/resources/{id}       # 更新资源
DELETE /api/v1/miniprogram/resources/{id}     # 删除资源
```

**前端实现方案**:

1. 创建统一的资源获取服务
2. 使用缓存机制减少API调用
3. 支持资源预加载和懒加载
4. 实现资源版本管理和灰度发布


**数据库设计建议**:
```sql
-- 小程序资源管理表
CREATE TABLE miniprogram_resources (
  id INT PRIMARY KEY AUTO_INCREMENT,
  resource_type VARCHAR(50) NOT NULL COMMENT '资源类型:banner/notice/service等',
  resource_data JSON NOT NULL COMMENT '资源数据(JSON格式)',
  sort_order INT DEFAULT 0 COMMENT '排序权重',
  is_enabled TINYINT DEFAULT 1 COMMENT '是否启用:0-禁用,1-启用',
  start_time DATETIME COMMENT '生效开始时间',
  end_time DATETIME COMMENT '生效结束时间',
  created_by INT COMMENT '创建人ID',
  updated_by INT COMMENT '更新人ID',
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX idx_type (resource_type),
  INDEX idx_enabled (is_enabled),
  INDEX idx_time (start_time, end_time)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='小程序资源管理表';
```

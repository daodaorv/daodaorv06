## 🎯 开发工作流程

### 📋 开发前准备
1. **阅读产品需求** → 了解业务功能和技术要求
2. **熟悉设计规范** → 掌握UI/UX设计标准
3. **配置开发环境** → 搭建uni-app开发环境
4. **代码规范学习** → 遵循团队编码标准

### 🛠️ 开发过程中
1. **组件优先级** → 内置组件 > uni-ui > 自定义组件
2. **代码质量** → TypeScript类型安全、错误处理、单一职责
3. **性能考虑** → 合理使用computed、避免重复渲染、优化加载
4. **跨端兼容** → 条件编译、upx单位、平台差异处理

### 📤 开发完成后
1. **代码审查** → 遵循代码审查检查清单
2. **功能测试** → 完整的业务流程测试
3. **性能测试** → 页面加载速度、交互响应时间
4. **兼容性测试** → 多平台、多设备测试

## ⚠️ 重要提醒

### 🔥 开发红线（绝对禁止）
- ❌ **绝不** 使用与uni-ui功能重复的自定义组件
- ❌ **绝不** 在uni-app项目中使用除uni-ui外的其他UI库
- ❌ **绝不** 开发小程序端前不阅读开发指导文档
- ❌ **绝不** 在代码中硬编码px单位，必须使用upx
- ❌ **绝不** 跳过跨端兼容检查
- ❌ **绝不** 提交未通过测试的代码

### ✅ 开发原则（必须遵循）
- ✅ **优先复用** → 充分利用uni-app和uni-ui官方组件
- ✅ **性能优先** → 考虑小程序环境下的性能优化
- ✅ **类型安全** → 使用TypeScript确保代码质量
- ✅ **用户体验** → 遵循设计规范，保证交互一致性
- ✅ **代码质量** → 遵循团队编码规范，编写可维护代码

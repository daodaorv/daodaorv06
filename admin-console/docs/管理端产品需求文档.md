## 1. 产品概述

### 1.1 产品定位

叨叨房车管理端是面向运营团队的综合性后台管理系统，为小程序端提供内容配置、规则管理、业务审核、数据分析等全方位支撑服务。

### 1.2 核心业务模块

| 业务模块 | 功能描述 | 目标用户 |
|----------|----------|----------|
| **核心业务管理** | 用户、车辆、门店、订单、托管、合作商、营地管理 | 全体运营人员 |
| **营销运营管理** | 价格策略、优惠券、营销活动、社区、客服管理 | 营销运营团队 |
| **分润财务管理** | 统一分润配置、分润记录、提现审核、财务报表 | 财务人员 |
| **系统管理** | 员工、权限、系统配置、内容资源管理 | 系统管理员 |

### 1.3 系统特点

- **七层角色权限体系**：平台管理员、区域经理、门店经理、门店员工、清洁工、维修工、客服
- **五种车辆所有权管理**：平台自有、自有车托管、购车托管、众筹托管、合作商
- **四种订单类型**：房车租赁、特惠租车、房车旅游、营地预订
- **四种分润类型**：推广分润、托管分润、合作商分润、员工分润
- **统一分润管理**：所有分润配置和记录集中在分润管理模块

## 2. 系统架构与角色权限

### 2.1 角色权限体系

#### 2.1.1 管理端角色（7种）

| 角色 | 角色代码 | 数据权限范围 | 主要职责 |
|------|----------|-------------|----------|
| **平台管理员** | `platform_admin` | 全平台数据 | 系统配置、全局管理、最高权限 |
| **区域经理** | `regional_manager` | 所辖区域所有门店数据 | 区域运营、跨门店协调 |
| **门店经理** | `store_manager` | 本门店数据 | 门店运营、员工管理 |
| **门店员工** | `store_staff` | 分配的任务数据 | 订单处理、客户服务 |
| **清洁工** | `cleaner` | 分配的整备任务 | 车辆清洁整备 |
| **维修工** | `mechanic` | 分配的维修任务 | 车辆维修保养 |
| **客服** | `customer_service` | 分配的工单数据 | 客户咨询、工单处理 |

#### 2.1.2 小程序端用户角色（5种）

| 角色 | 角色代码 | 说明 |
|------|----------|------|
| **普通注册用户** | `normal` | 基础用户，可租车、预订营地等 |
| **PLUS用户** | `plus` | 付费会员，享受会员权益 |
| **自有车托管用户** | `hosting_own` | 托管自有车辆的车主 |
| **购车托管用户** | `hosting_purchase` | 通过平台购车托管的车主 |
| **众筹托管用户** | `hosting_crowdfunding` | 参与众筹的用户 |

**重要说明**：
- 用户可同时拥有多个角色（如：PLUS用户 + 自有车托管用户）
- PLUS会员是唯一的付费会员类型，**不存在**银卡/金卡/铂金等级

### 2.2 数据权限控制

| 角色 | 用户数据 | 车辆数据 | 订单数据 | 财务数据 |
|------|----------|----------|----------|----------|
| 平台管理员 | 全部 | 全部 | 全部 | 全部 |
| 区域经理 | 区域内 | 区域内 | 区域内 | 区域内 |
| 门店经理 | 门店内 | 门店内 | 门店内 | 门店内 |
| 门店员工 | 分配的 | 分配的 | 分配的 | 无 |
| 清洁工 | 无 | 分配的 | 无 | 无 |
| 维修工 | 无 | 分配的 | 无 | 无 |
| 客服 | 工单相关 | 无 | 工单相关 | 无 |

### 2.3 登录与注册流程

#### 2.3.1 登录功能

**登录方式**：手机号 + 密码

**登录页面字段**：
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 手机号 | input | 是 | 11位手机号 |
| 密码 | password | 是 | 6-20位密码 |
| 记住密码 | checkbox | 否 | 7天内免登录 |

#### 2.3.2 注册申请流程

**申请入口**：登录页"申请注册"按钮

**身份类型选择**：
| 身份类型 | 说明 | 审核后角色 |
|----------|------|------------|
| 平台员工 | 申请成为平台员工 | 门店员工 |
| 清洁工 | 申请成为清洁工 | 清洁工 |
| 维修工 | 申请成为维修工 | 维修工 |
| 客服 | 申请成为客服 | 客服 |

**申请表单字段**：
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 手机号 | input | 是 | 11位手机号 |
| 真实姓名 | input | 是 | 2-20个字符 |
| 身份证号 | input | 是 | 18位身份证号 |
| 身份类型 | select | 是 | 选择申请的身份 |
| 期望职位 | input | 条件必填 | 平台员工需填写 |
| 服务区域 | select | 条件必填 | 清洁工/维修工需选择 |
| 申请说明 | textarea | 否 | 申请理由和相关经验 |

**审核流程**：
1. 用户提交申请
2. 管理员在"员工管理 → 注册审核"中审核
3. 审核通过后自动分配角色和权限
4. 系统发送审核结果通知
5. 用户可使用手机号登录

**审核时效**：3个工作日内完成

**页面路径**：`/partners/orders`

**功能描述**：管理合作商订单信息，填写订单号和价格，自动计算差价

#### 8.4.1 手动订单对接流程

1. 用户在小程序下单（选择合作商车辆）
2. 生成叨叨订单（订单号、订单价格）
3. 管理员手动到合作商网站下单
4. 获取合作商订单号和订单价格
5. 在管理端填入合作商订单信息
6. 系统自动计算差价

#### 8.4.2 差价计算公式

```
订单差价 = 叨叨订单价格 - 合作商订单价格
```

## 21. 业务流程定义

### 21.1 房车租赁核心流程

#### 21.1.1 用户下单流程

```
用户选择车辆 → 选择租赁时间 → 选择取还车方式 → 填写驾驶人信息
    ↓
计算费用（租金+保险+增值服务+异地还车费）
    ↓
应用优惠券 → 选择支付方式 → 支付
    ↓
生成订单（状态：待确认）
    ↓
平台确认（状态：已确认）→ 等待取车
```

#### 21.1.2 取车流程

```
用户到达门店/等待送车 → 核验身份证+驾驶证
    ↓
车辆检查（外观、里程、油量、设备）→ 签署租赁协议
    ↓
收取押金（传统押金/信用免押）→ 交付车辆
    ↓
订单状态更新为"进行中"
```

#### 21.1.3 还车流程

```
用户到达还车点/等待上门取车 → 车辆检查
    ↓
核对里程、油量、车况 → 确认是否有损坏/违章
    ↓
计算额外费用（超里程、超时、损坏、清洁等）
    ↓
退还押金（扣除额外费用）→ 订单状态更新为"已完成"
    ↓
邀请用户评价
```

### 21.2 托管业务流程

#### 21.2.1 自有车托管申请流程

```
车主提交托管申请（车辆信息+照片+行驶证）
    ↓
管理员审核（1-3个工作日）
    ├── 通过 → 签署托管协议 → 车辆入库 → 分配角色"自有车托管用户"
    └── 拒绝 → 通知车主原因
```

#### 21.2.2 托管车辆收益分配流程

```
托管车辆产生订单 → 订单完成
    ↓
计算订单收益 → 按分成比例计算车主收益
    ↓
收益入账（车主钱包）→ 车主可申请提现
```

#### 21.2.3 车主自用申请流程

```
车主提交自用申请（时间、取还车门店）
    ↓
系统检查：
├── 车辆状态是否可用
├── 是否有订单冲突
├── 是否有维保计划冲突
    ↓
管理员审核（1个工作日）
    ├── 通过 → 计算费用（服务费+异地还车费）→ 车主确认
    └── 拒绝 → 通知车主原因
```

### 21.3 众筹托管流程

#### 21.3.1 众筹项目流程

```
平台发布众筹项目（车型、总份额、单价、目标金额）
    ↓
用户认购份额 → 支付份额款项
    ↓
达到目标金额 → 项目成功 → 购车入库
    ↓
车辆运营 → 产生收益 → 按份额比例分配
```

#### 21.3.2 份额转让流程

```
份额持有人发起转让（份额数量、转让价格）
    ↓
平台审核 → 上架交易
    ↓
买家购买 → 支付款项 → 份额过户
    ↓
更新份额持有记录 → 通知双方
```

### 21.4 合作商订单流程

```
用户在小程序选择合作商车辆 → 下单支付
    ↓
生成叨叨订单（订单号、订单价格）
    ↓
管理员手动到合作商网站下单 → 获取合作商订单号和价格
    ↓
在管理端填入合作商订单信息
    ↓
系统自动计算：
├── 订单差价 = 叨叨订单价格 - 合作商订单价格
├── 门店分润 = 订单差价 × 门店分润比例
└── 平台收益 = 订单差价 × (1 - 门店分润比例)
```

### 21.5 分润结算流程

#### 21.5.1 推广分润流程

```
A用户推荐B用户注册 → B用户下单并完成
    ↓
计算A的一级分润 → 入账A的钱包
    ↓
如果A是被C推荐的 → 计算C的二级分润 → 入账C的钱包
```

#### 21.5.2 提现流程

```
用户申请提现（金额、提现方式）
    ↓
系统校验：
├── 余额是否充足
├── 是否满足最低提现金额
├── 是否超过单日/单月限额
    ↓
管理员审核
    ├── 通过 → 执行打款 → 更新钱包余额
    └── 拒绝 → 通知用户原因 → 退回余额
```

### 21.6 员工注册审核流程

```
申请人在移动端提交注册申请（身份类型、个人信息）
    ↓
管理员在"员工管理 → 注册审核"中审核（3个工作日）
    ├── 通过 → 创建员工账号 → 分配角色和权限 → 通知申请人
    └── 拒绝 → 通知申请人原因
```

### 22.5 v3.0.0 优化说明

#### 22.5.1 消除的功能重复

| 原重复功能 | 原位置 | 优化后位置 |
|------------|--------|------------|
| Banner管理 | 内容管理 + 小程序资源 | 统一到内容资源管理模块 |
| 内容管理 + 小程序资源 | 两个独立菜单 | 合并为"内容资源管理"单一入口 |
| 分润配置 | 合作商管理 + 分润管理 | 统一到分润管理模块 |
| 托管收益/分润 | 托管管理 + 分润管理 | 托管管理负责运营，分润管理负责分润 |

#### 22.5.2 菜单结构优化

- **内容资源管理**：合并原"内容管理"和"小程序资源"为单一菜单入口
- 移除"房车租赁"模块中的快捷入口（避免路由重复）
- 简化托管管理菜单层级（从4层减少到3层）
- 系统管理模块不再包含小程序资源管理

## 🎯 核心规范概览

### 胶水编程优先

- 能复用不造轮子，能连接不重写
- 优先使用成熟的开源库和官方 SDK
- 自定义代码仅用于：组合、调用、封装、适配

### 工作流程

1. **研究阶段** - 检查现有实现，搜索最佳实践
2. **计划阶段** - 列出文件清单，说明实现方案
3. **实现阶段** - 遵循代码风格，完整错误处理

### 质量红线

- ✅ 必须通过 Linter（零警告零错误）
- ✅ 必须通过所有测试
- ✅ 必须完成代码格式化
- ✅ 必须通过类型检查（TypeScript）

## 🔧 Backend 特定规范

### API 设计原则

- **路由命名**：必须使用名词，复数形式
- **RESTful 标准**：GET/POST/PUT/PATCH/DELETE
- **响应格式**：统一的 JSON 格式
- **HTTP 状态码**：严格遵循标准

### 技术栈

- Node.js 18.18.0
- Express 4.18.2
- TypeScript 5.1.6
- Sequelize 6.32.1
- MySQL 8.0.35
- Redis 7.2.3

### 开发命令

```bash
cd backend
npm run dev          # 端口 3001
npm run lint         # ESLint 检查
npm test             # 运行测试
```text

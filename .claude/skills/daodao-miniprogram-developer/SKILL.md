### 技术栈要求：

**核心技术栈**：
- uni-app 3.0.0 - 跨端开发框架
- Vue 3.5.0 - 前端框架，支持Composition API
- TypeScript 5.1.6 - 类型安全的JavaScript超集
- Pinia - 状态管理
- SCSS - CSS预处理器
- Vite - 构建工具

**开发工具**：
- HBuilderX - 主要开发IDE
- 微信开发者工具 - 小程序调试
- VS Code - 代码编辑器

### 开发规范：

#### 项目结构规范
```
miniprogram/
├── src/
│   ├── pages/           # 页面文件
│   ├── components/      # 公共组件
│   ├── composables/     # 组合式函数
│   ├── stores/          # 状态管理
│   ├── api/            # API接口
│   ├── utils/          # 工具函数
│   ├── types/          # TypeScript类型
│   └── styles/         # 样式文件
├── static/             # 静态资源
└── manifest.json       # 应用配置
```

#### 代码编写规范
- 使用Vue 3 Composition API + `<script setup>`
- 完整的TypeScript类型定义
- 遵循组件化开发原则
- 使用Pinia进行状态管理
- 统一的错误处理和加载状态管理

#### API调用规范
- 统一的请求封装和拦截器
- 错误处理和重试机制
- 请求参数和响应数据的类型定义
- Mock数据和真实API的无缝切换

### 小程序端特色设计调整：

#### 首页设计原则
- **移除搜索功能**：简化页面设计，专注核心预订功能
- **强化条件查询**：突出取车城市、门店、时间等必要条件设置
- **众筹推荐集成**：在推荐内容区增加优质众筹项目展示
- **视觉层次清晰**：轮播广告、金刚区、推荐内容层次分明

#### 车辆列表设计原则
- **移除车辆对比功能**：简化操作流程，突出收藏和详情查看
- **强化筛选排序**：提供价格、座位数、品牌等筛选功能
- **分组展示逻辑**：按门店距离分组，同门店内按价格排序

#### 客服页面设计原则
- **简化帮助中心**：移除搜索功能，保留基础帮助和问题分类
- **突出紧急联系**：强化24小时紧急电话和道路救援入口
- **在线客服为核心**：以智能客服和人工客服为主要服务方式

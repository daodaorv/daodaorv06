# 叨叨房车原型图设计文档

**文档版本**: v2.0 | **更新时间**: 2025-11-23 | **状态**: 执行中

## 1. 设计目标

本原型设计的核心目标是输出**高保真、可直接移植**的 HTML 原型。
*   **高保真**: 严格遵循 `Docs/小程序UI设计指南.md` 的视觉规范（配色、圆角、间距）。
*   **可移植**: 代码结构清晰，组件化拆分，方便后续直接复制到 uni-app 项目中。
*   **全覆盖**: 覆盖小程序端和移动管理端的核心业务流程。

## 3. 技术实现规范

### 3.1 基础框架
*   **HTML5**: 语义化标签。
*   **Tailwind CSS**: 用于快速布局和原子样式 (使用 CDN 引入，方便单文件预览)。
*   **Custom CSS**: 定义项目特有的设计令牌 (Design Tokens)。

### 3.2 样式变量 (对应 `variables.scss`)
在 `assets/css/variables.css` 中定义：
```css
:root {
    /* 品牌色 */
    --primary-orange: #FF9F29;
    --primary-orange-light: rgba(255, 159, 41, 0.1);
    --member-purple: #8860D0;
    
    /* 辅助色 */
    --success-green: #67C23A;
    --error-red: #FF4D4F;
    --info-blue: #4B91FF;
    
    /* 文字色 */
    --text-primary: #1A1A1A;
    --text-secondary: #666666;
    --text-tertiary: #999999;
    
    /* 背景色 */
    --bg-page: #F7F8FA;
    --bg-card: #FFFFFF;
}
```

### 3.3 组件化策略
虽然是 HTML 原型，但应通过注释和结构清晰地划分组件，例如：
```html
<!-- Component: VehicleCard -->
<div class="vehicle-card bg-white rounded-lg p-4 shadow-sm">
    ...
</div>
<!-- End Component -->
```

## 5. 移植指南
1.  **HTML 结构**: 直接复制 `<body>` 内的核心内容到 Vue `<template>` 中。
2.  **Tailwind 类名**: 大部分可直接保留，部分需转换为 uni-app 的 `class` 或 `style`。
3.  **图片资源**: 替换为项目中的 `@/static/` 路径。
